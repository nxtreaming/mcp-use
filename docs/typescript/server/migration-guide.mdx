<Warning>
This migration guide applies to **mcp-use TypeScript SDK version 1.5.0 and later**. If you're using an earlier version, these changes do not apply.
</Warning>


## Removed Components

### WidgetDebugger → WidgetControls

The `WidgetDebugger` component has been removed. Its functionality has been integrated into `WidgetControls`.

#### Before (WidgetDebugger)

```tsx
import { WidgetDebugger } from 'mcp-use/react';

function MyWidget() {
  return (
    <WidgetDebugger>
      <div>My widget content</div>
    </WidgetDebugger>
  );
}
```

#### After (WidgetControls)

```tsx
import { WidgetControls } from 'mcp-use/react';

function MyWidget() {
  return (
    <WidgetControls debugger>
      <div>My widget content</div>
    </WidgetControls>
  );
}
```

**Key Changes:**
- `WidgetDebugger` → `WidgetControls` with `debugger` prop
- Same debug overlay functionality
- Additional view controls available via `viewControls` prop

#### Recommended: Use McpUseProvider

For most cases, use `McpUseProvider` which includes `WidgetControls` automatically:

```tsx
import { McpUseProvider } from 'mcp-use/react';

function MyWidget() {
  return (
    <McpUseProvider debugger>
      <div>My widget content</div>
    </McpUseProvider>
  );
}
```

### WidgetFullscreenWrapper → McpUseProvider + WidgetControls

The `WidgetFullscreenWrapper` component has been removed. Its functionality is now provided by `McpUseProvider` and `WidgetControls`.

#### Before (WidgetFullscreenWrapper)

```tsx
import { WidgetFullscreenWrapper } from 'mcp-use/react';

function MyWidget() {
  return (
    <WidgetFullscreenWrapper>
      <div>My widget content</div>
    </WidgetFullscreenWrapper>
  );
}
```

#### After (McpUseProvider)

```tsx
import { McpUseProvider } from 'mcp-use/react';

function MyWidget() {
  return (
    <McpUseProvider viewControls>
      <div>My widget content</div>
    </McpUseProvider>
  );
}
```

**Key Changes:**
- `WidgetFullscreenWrapper` → `McpUseProvider` with `viewControls` prop
- Fullscreen and PIP controls are now part of `WidgetControls`
- `McpUseProvider` also includes other necessary providers (ThemeProvider, BrowserRouter, ErrorBoundary)

#### Complete Migration Example

**Before:**

```tsx
import { WidgetDebugger, WidgetFullscreenWrapper } from 'mcp-use/react';

function MyWidget() {
  return (
    <WidgetFullscreenWrapper>
      <WidgetDebugger>
        <div>My widget content</div>
      </WidgetDebugger>
    </WidgetFullscreenWrapper>
  );
}
```

**After:**

```tsx
import { McpUseProvider } from 'mcp-use/react';
import { AppsSDKUIProvider } from '@openai/apps-sdk-ui/react';
import { Link } from 'react-router-dom';

function MyWidget() {
  return (
    <McpUseProvider debugger viewControls autoSize>
      <AppsSDKUIProvider linkComponent={Link}>
        <div>My widget content</div>
      </AppsSDKUIProvider>
    </McpUseProvider>
  );
}
```

## Benefits of Migration

### Unified Provider

`McpUseProvider` combines all common setup:
- ✅ StrictMode (development checks)
- ✅ ThemeProvider (theme management)
- ✅ BrowserRouter (routing support)
- ✅ WidgetControls (debug + view controls)
- ✅ ErrorBoundary (error handling)
- ✅ Auto-sizing (optional, for dynamic content)

### Better Organization

- Single component instead of multiple wrappers
- Clear prop-based configuration
- Consistent patterns across widgets

### Enhanced Features

- Auto-sizing support with `autoSize` prop
- Better dev mode detection
- Improved error handling
- More flexible control positioning

## Step-by-Step Migration

1. **Replace WidgetDebugger**:
   ```tsx
   // Old
   <WidgetDebugger>...</WidgetDebugger>
   
   // New
   <WidgetControls debugger>...</WidgetControls>
   // Or use McpUseProvider
   <McpUseProvider debugger>...</McpUseProvider>
   ```

2. **Replace WidgetFullscreenWrapper**:
   ```tsx
   // Old
   <WidgetFullscreenWrapper>...</WidgetFullscreenWrapper>
   
   // New
   <McpUseProvider viewControls>...</McpUseProvider>
   ```

3. **Combine Both**:
   ```tsx
   // Old
   <WidgetFullscreenWrapper>
     <WidgetDebugger>...</WidgetDebugger>
   </WidgetFullscreenWrapper>
   
   // New
   <McpUseProvider debugger viewControls>...</McpUseProvider>
   ```

4. **Add AppsSDKUIProvider** (if using OpenAI Apps SDK UI components):
   ```tsx
   <McpUseProvider debugger viewControls>
     <AppsSDKUIProvider linkComponent={Link}>
       <MyWidget />
     </AppsSDKUIProvider>
   </McpUseProvider>
   ```

5. **Enable Auto-sizing** (optional, for dynamic content):
   ```tsx
   <McpUseProvider debugger viewControls autoSize>
     <AppsSDKUIProvider linkComponent={Link}>
       <MyWidget />
     </AppsSDKUIProvider>
   </McpUseProvider>
   ```

## Import Changes
